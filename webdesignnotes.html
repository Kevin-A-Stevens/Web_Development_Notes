Seven steps to building a web site

  1. Define
    1. Who is the website for
    2. What is the website for
    3. What is the main goal/purpose
    4. Who is the target audience
    5. Be specific. These can come from the client
  2. Plan
    1. Gathering the content: copy(text), images, videos, etc
    2. Content is usually provided by client
    3. find free images
    4. Charge if not easy
    5. Plan out the site map for big sites
    6. Based on content, plan what sections the site needs
    7. Define the website personality
  3. Sketch
    1. Think about what components and layout patterns you need
    2. Get ideas and sketch with pen and paper or by using software such as Figma
    3. This is an iterative process
    4. Experiment with different ideas, components, and layouts
    5. Don't need to sketch everything and don't make it perfect
    6. Content should guide the design
  4. Design and Build
    1. Use decisions, content, and sketches to design using HTML and CSS
    2. Design the actual visual styles you decided on and website personality or personalities
    3. Use clients branding if it exists (colors, typography, icons, etc)
  5. Test and Optimize
    1. Make sure website works in all major browsers (Chrome, Safari, Edge, Firefox)
    2. Test on actual mobile devices - not just dev tools
    3. Optimize all images in dimensions and file size
    4. Fix accessability problems (color contrast issues
    5. Run the Lighthouse performance test in Chrome Dev Tool and try to fix reported issues
    6. Think about Search Engine Optimization (SEO)
  6. Launch
    1. Share it with the world
    2. Netlify has a free plan but there are others
    3. Choose and buy a great domain name, one that represents the brand well, memorable, easy to write
  7. Maintain and Update
    1. Keep the website content updated over time
    2. Create a monthly maintenance contract with your client
    3. Install analytics software such Google Analytics or Fathom
    4. A blog is updated regularly is a good way to keep users coming back, and is also good for SEO

Website Personalities - Choose one

  1. Serious/Elegant = thin serif typefaces, golden or pastel colors, big high quality images
  2. Minimalist/Simple = small to medium sans-serif, black text lines and few images/icons
  3. Plain/Neutral = neutral small typefaces, very structured layout. Big corporations.
  4. Bold/Confident = Big and bold typography paired with big/bright colored blocks.
  5. Calm/Peaceful = care. Calming pastel colors. soft serif headings matching images/illustrations.
  6. Startup/Upbeat = startup, featured medium sized sans-serif typefaces. light-grey text/backgrounds
  7. Playful/Fun = colorful/rounds designs. Hand-drawn/creative elements, animations, and fun language

Rules of Website Design

Rule #1 Typography

  1. Typography - Choose one or two, not more
  2. Use https://fonts.google.com/ or https://type-scale.com/
  3. Use only good/popular typefaces. Limit # of typefaces to 2
  4. Sans-serif: inter, open sans, roboto, montserrat, work sans lato
  5. Sans: merriweather, aleo, playfair display, cormorant, cardo, lora
  6. Choose font-size, limit choices, use type-scale
  7. normal text: 16px to 32px
  8. long text: 20px or bigger such as a long blog post
  9. headlines: big like 50px+ and bold 600+. Depends on personality
  10. any text: no font weight under 400 (regular)
  11. Use less than 75 characters per line
  12. normal sized text, use line height between 1.5 and 2
  13. big text go below 1.5 line height
  14. decrease letter spacing in headlines if it looks unnatural
  15. Experiment with all caps for short titles. Small and bold increases letter spacing
  16. Don't center long text blocks. Small blocks are fine

Rule #2 Colors

  1. Use https://yeun.github.io/open-color/
  2. https://coolors.co/ -- More > Contrast Checker
  3. Choose a color that matches your websites personality - See above
    1. Red - symbolizes power, passion, excitement
    2. orange - less aggressive, happiness, cheerfulness, creativity
    3. yellow - joy, brightness, intelligence
    4. green - represents harmony, nature, health
    5. blue - peace, trustworthiness, professionalism
    6. purple - wealth, wisdom, magic
    7. pink - romance, care, affection
    8. brown - nature, durability, comfort
    9. black - power, elegance, minimalism, grief, sorrow
  4. Choose a good color tone - Open color, tailwind CSS, flat UI Colors
  5. Choose a Main color and a Grey color (text is usually grey color)
  6. Choose a secondary color for an accent color if you want. Needs to be a relationship with the main color
  7. For diversity, create lighter and darker versions of your chosen colors - palleton.com, coolors
  8. Main color - draw attention to the main element of a page such as a button and/or logo
  9. Use colors to add accents or make components and sections stand out
  10. Use colors strategically in images and illustrations
  11. For dark backgrounds, try to use a tint of teh background (lighter version) for text
  12. Text should not be completely black - Use a shade of grey instead
  13. Don't make your text too light

Rule #3 Images and Illustrations

  1. Use https://unsplash.com/ or https://squoosh.app/
  2. Product Photos - illustrate the product
  3. Storytelling photos - helps to convey the message of your website (most important)
  4. Illustrations - abstract way of doing storytelling
  5. Patterns - background of an entire sections or behind images to add a visual image detail. Adds creativity
  6. Prefer original images you take yourself or by a professional photographer if budget allows
  7. Try to show real people to trigger emotions
  8. If necessary, crop your images to fit your message
  9. Combine photos, illustrations, and patterns
  10. Darken or brighten the image where the text is
  11. Position text into a neutral area
  12. Put text in box, perhaps putting opacity on the box
  13. For high res screens, make image dimensions 2x as big as their display size
  14. Compress images for a lower filer size and better performance

Rule #4 Icons

  1. Use icons only from one icon pack. Don't mix
  2. Use https://heroicons.com/
  3. Use a good icon pack: phosphor icons, ionicons, icons8
  4. You can use emojis as well
  5. Use only one item pack
  6. Use only SVG icons or icon fonts (do not use .jpg or.png for icons as they do not scale)
  7. Adjust to website personality (roundness, weight, filled, outlined depend on typography)
  8. Use icons to provide visual assistance
  9. Use icons to build feature blocks - describe features of a service
  10. Use icons associated with actions, and label them
  11. Use icons as bullet points
  12. Keep icons neutral. Use same color as text. To draw more attention, use a different color
  13. Use icons that make sense and fit the text or action you want to convey
  14. Don't make icons larger than what they were designed for

Rule #5 Shadows

  1. Shadows create depth and adds a 3d element to our design
  2. Use shadows only if it makes sense for the website personality
  3. The more playful a design gets, the more shadows you can use
  4. Less shadows for more serious and elegant the design
  5. Go light on shadows and don't go too dark
  6. Use shadows on elements you want to draw attention to such as button
  7. Use medium sized shadows for larger areas that should stand out more like an entire section or cards
  8. Use large shadows for elements you want to float above the interface like navigation or pop ups
  9. Change shadows on mouse interaction like click or hover

Rule #6 Border Radius

  1. Increase the playfulness and fun of the design
  2. Typeface can determine how much border-radius and certain typography is more round than others
  3. Buttons use border radius the most
  4. Images and icons are also common to have border-radius

Rule #7 White Space

  1. Makes the design clean, modern, and polished
  2. Use whitespace between sections and between groups of elements
  3. Usually need more white space vertically than horizontally between elements
  4. Inside groups of elements, try to use whitespace instead of lines
  5. Use the law of proximity - groups of elements belong together, the closer they should be
  6. Start with a lot of whitespace, then remove whitespace until it looks right
  7. Match other design choices - huge text should equal more whitespace
  8. Try using a hard rule such as multiples of 16px for all margins and paddings on a page
  9. 2, 4, 8, 12, 16, 32, 48, 64, 80, 96, 128

Rule #8 Visual Hierarchy

  1. Which element of a design are the the most ones
  2. Define a path for users to guide them through the page
  3. Position important elements closer to the top of the page where they get more attention
  4. First call-to-action also as close as possible to the top of the page
  5. White space creates separation so use whitespace strategically to emphasize elements
  6. Text - use font size, font weight, color, and whitespace to convey importance
  7. Emphasize titles, sub-titles, links, buttons, data points, icons
  8. De-emphasize less important text, labels, secondary/additional information
  9. Components to stand out: testimonials, call-to-action sections, highlight sections, preview cards, forms, pricing tables, important rows/columns in tables

Rule #9 User Experience (UX)

  1. Don't design complicated layouts. Use patterns that users know
  2. Make you call-to-action the most prominent element, and make the text descriptive
  3. Use blue text and underline text only for links
  4. Animations should have a purpose and be fast. Between 200ms and 500ms
  5. Forms - align labels and fields in a single vertical line gto make the form easier to scan
  6. Offer users good feedback for all actions: form errors, form success, etc
  7. Law of locality - place buttons where they will create an effect. Does not apply to call-to-action
  8. Write a descriptive headline, keyword-focused on your main page
  9. Only include relevant information. Everything presented should have a very clear purpose
  10. Use simple words and avoid technical jargon
  11. Break up long text with sub-headings, images, block quotes, bullet points, etc

Rule #10 Building a Webpage

  1. Build your elements - headers, paragraphs, etc you use to present the information you want
  2. Elements - text, buttons, images, input elements, tags
  3. Build your components by combining elements
  4. Components - Breadcrumbs, pagination, alert/status bars, statistics, gallery, feature box, preview/portfolio cards, accordion, tabs, carousel, customer testimonials, logos, featured-in logos, steps, forms, tables, pricing tables, modal windows, etc
  5. Section components - navigation, hero section, footer, call-to-action section, feature row
  6. Combine components into a common layout pattern
  7. Layout patterns - Row of boxes/cards, grid of boxes/cards, Z-pattern, F-pattern, Single column, Sidebar, Multi-column magazine, Asymmetry/Experimental
  8. Assemble a final page from your layout

The Website Personality Framework

  1. Choose how you want the website to appear to users
  2. Choose one of the website personalities accordingly
  3. Apply personality traits to each of the design ingredients
    1. Serious/Elegant - thin serif typefaces, golden or pastel colors, big high quality images. Usually no shadows and no border radius and no icons
    2. Minimalist/Simple - use for portfolios. boxy, sans-serif typefaces, black or dk grey color, just one main color, few images, small black icons
    3. Plain/Neutral - sans-serif, nothing too bright, blues and black, images are frequently used. simple icons may be used, usually no shadows or border radius
    4. Bold/Confident - boxy/squared typefaces, big and bold. bright colors and big color sections. lots of big images. usually not used are icons, shadows, or border-radius
    5. Calm/Peaceful - soft serif typefaces, light orange or yellows, browns, greens, blues. images have a lot of people and match color palette. Icons are frequent. Shadows used sparingly, some border-radius us usual
    6. Startup/Upbeat - usually sans-serif typography, green, blue, purple for primary, light grey for background, common to use gradient. Images are always used and 3d illustrations. subtle shadows, border-radius is common and icons are also frequent
    7. Playful/Fun - round creative typography and sans-serif. Multiple colors. Images are 3d and illustrations. icons, shadows, and border-radius are frequent
  4. Can mix two designs by using one main personality with a secondary personality
